# 搜索规则说明

岁影 v1 的搜索规则参考 `hunhepan-rule-template-main` 的事件模型，并在此基础上增加影视实体识别与结果重排：

- `MediaCandidate`：识别到的影视条目，例如电视剧《生命树》。
- `SearchPlan`：由原始输入生成的资源搜索词、包含关键词和排除关键词。
- `onSearch` 对应 `search_resources` 命令中的来源搜索适配。
- `onInfo` 对应 `get_resource_detail` 命令中的详情解析。
- `manifest.json` 中的来源名称、分组、启用状态，对应 Rust 里的 `SearchSource`。

## 搜索流程

```text
用户输入 -> 影视实体识别 -> 资源搜索词生成 -> 多来源召回 -> 结果重排 -> 分组展示
```

例如输入 `胡歌 生命树`：

```text
识别影视条目：生命树 / 电视剧 / 2026 / 胡歌、杨紫 / 爱奇艺 / Born to be alive
搜索词：生命树、生命树 2026、生命树 全集、生命树 更新、生命树 40集、生命树 杨紫 胡歌、生命树 夸克、生命树 阿里云盘、生命树 爱奇艺、Born to be alive、胡歌 生命树
```

## 资源池结构

岁影当前按四层资源池召回：

| 资源池 | 配置方式 | 说明 |
| --- | --- | --- |
| PanSou 深度池 | 设置页 `pansouEndpoints` | 支持多个 PanSou endpoint，配置 `src`、`channels`、`plugins`、`cloudTypes`、缓存与刷新。搜索时会把影视条目的中文名、英文名、年份、主演、平台等信息放入 `ext`。 |
| CMS V10 源池 | 设置页 `cmsSources` | 支持文本或 JSON 批量导入，健康检测调用 `?ac=videolist&wd=生命树`，搜索调用 `?ac=videolist&wd=关键词`，详情地址使用 `?ac=detail&ids=ID`。 |
| Torznab/Newznab 索引器 | 设置页 `indexers` | 仅接用户自有或已授权索引器。电视剧优先 `t=tvsearch`，电影优先 `t=movie`，无结果时兜底 `t=search`。 |
| 本地规则插件池 | `rules/sources/*.json` | 描述默认公开页面源和需配置源元数据，具体请求和解析仍在 Rust 适配器内执行。 |

## 当前来源

| 来源 | 状态 | 说明 |
| --- | --- | --- |
| PanSou | 可配置 | 配置 endpoint 后调用 `/api/search`，支持网盘类型、频道、插件、缓存和强制刷新。 |
| CMS V10 | 可配置 | 配置苹果 CMS V10 接口后调用 `?ac=videolist&wd=关键词`。 |
| Torznab/Newznab | 可配置 | 配置外部索引器后解析 RSS XML item，结果只展示和跳转。 |
| TMDB | 可配置 | 配置 API Key 后调用 `search/multi` 增强影视候选识别，不直接提供资源链接。 |
| 混合盘 | 启用 | 调用开放搜索接口，详情可直接返回跳转地址。 |
| 皮卡搜索 | 启用 | 页面规则解析，结果多数直接是网盘链接。 |
| 阿里搜 | 启用 | 页面规则解析，详情阶段尝试补充跳转信息，失败时回退到来源详情页。 |
| 咔帕搜索 / 捕娱兔 / 学霸盘 / 夸克搜 / 云盘资源共享站 / 我的小站 / 盘趣多 | 启用 | 默认公开页面源；搜索多关键词、多页，详情阶段尽量提取网盘链接。 |
| 千帆 / 易搜 / 夸克盘搜 | 需配置或禁用 | 需要 code、cookie 或当前直连不稳定，默认不参与搜索。 |

## 本地规则文件

默认规则文件：

```text
rules/sources/default.json
```

规则只描述来源元数据，不保存用户 token：

```json
{
  "id": "hunhepan",
  "name": "混合盘",
  "group": "公开页面源",
  "kind": "hunhepan",
  "enabled": true,
  "description": "混合盘开放搜索接口，详情可直接得到跳转链接。",
  "healthScore": 64,
  "status": "ready"
}
```

`kind` 对应 Rust 内的适配器分支。后续扩展 Nullbr、MoviePilot 插件类 API 时，应新增 `custom-http-json` 这类通用规则，不把具体服务写死进代码。

## 失败处理

- 多来源并发执行，单个来源失败只会更新该来源状态，不影响其他来源展示。
- UI 会展示每个来源的成功、空结果、失败或禁用状态。
- UI 会按公开页面源、需配置源、PanSou 深度池、CMS 源池、外部索引器统计来源覆盖，暴露资源池是否过浅。
- 外部站点结构变化时，优先修复对应来源适配器，不影响前端契约。
- PanSou endpoint 或 CMS V10 接口未配置时，来源显示为未配置，不影响现有页面源。

## 结果重排

- 强提升：片名精确命中、包含年份、包含全集/更新/集数、主演、平台、链接有效、PanSou 来源。
- 中提升：包含主演、平台、别名、清晰度、合集、CMS 源或外部索引器。
- 强降权：音乐、电子书、广告、课程、预告、花絮、纪录片误命中，以及《生命之树》《生命树物语》、游戏、免安装版、课件、生物植物类噪声。
- 《生命树》高可信结果必须“片名命中 + 影视信号”，影视信号包括 `2026`、`40集`、`全集`、`更新`、`电视剧`、`杨紫`、`胡歌`、`爱奇艺` 或集数格式。
- 去重：按最终链接、标题和资源 hash 合并重复结果，减少多来源重复刷屏。

## 合规边界

- 不内置灰色资源站、tracker、公开 PanSou 服务或 CMS 源；默认公开页面源只做搜索、解析和跳转。
- Torznab/Newznab 只面向用户自有或有权限的索引器。
- 岁影只做资源发现、索引适配和跳转，不下载、不破解、不绕过付费、DRM 或访问控制。

## 验证关键词

- `胡歌 生命树`：应识别电视剧《生命树》，并命中盘趣多的《生命树》2026 / 全40集网盘结果。
- `斯巴达克斯`：混合盘已验证可以返回结果。
- `生命树`：应优先展示电视剧《生命树》网盘结果；《生命树物语》《生命之树》不能进入高可信区。
